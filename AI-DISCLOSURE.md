# AI Disclosure & Provenance

This repository is a human-directed, AI-assisted project.

## Authorship Boundaries

### Human Ownership

The project direction, problem framing, and operational intent for Wazuh Docker Config Manager were set by the repository author.

That includes:

- the decision to build a local admin dashboard for Wazuh Docker operations
- the focus on practical configuration management instead of broad platform orchestration
- the UX and workflow direction around rules, decoders, config, backups, lists, agents, and container controls
- the architectural decision to combine Docker-based file operations with Wazuh REST API interactions

In other words: the product intent and engineering direction came from the human author.

### AI Assistance

AI tools were used as development support during implementation.

That assistance included:

- code generation and code editing
- refactoring support
- debugging and iteration
- documentation drafting and cleanup
- UI copy and workflow refinement

AI was used as an interactive tool, not as the autonomous owner of the project direction.

## Models / Tools Used

Development has involved AI-assisted coding sessions across more than one tool over time.

Known examples include:

- OpenAI Codex / GPT-based coding assistance
- Claude-based coding assistance

This should be read as a good-faith disclosure, not a complete forensic log of every individual prompt or session.

## Review and Responsibility

AI-assisted output was reviewed, accepted, rejected, modified, or integrated by a human before being retained.

Final responsibility for:

- project decisions
- design direction

- the integrated final form of the work

rests with the human author.

This repository is presented as a human-authored project developed with AI assistance.

The author claims authorship over the repository as a whole, including:
